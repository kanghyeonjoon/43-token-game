# AI 직원 오케스트레이션 가이드 (v0.10)

> 케어루프 CRM에 **AI 직원 팀**을 붙이는 기능입니다. 오케스트레이터(AI 실장)가 병원 데이터를 조회하고
> 전문 AI 직원 4명에게 업무를 배분해, 브리핑·문안 작성·컴플라이언스 검수·데이터 분석을 처리합니다.
> 구현: `db/edge-ai-employee.ts` (Supabase Edge Function) + 대시보드 **AI 직원** 화면.

---

## 1. 왜 "AI 직원"인가 — 설계 철학

케어루프의 자동화 시나리오 12개(기획서 5장)는 **규칙 기반**입니다. 트리거가 명확한 반복 업무(D+5 경과체크,
재안내)는 규칙이 가장 싸고 정확합니다. AI 직원은 그 위에 얹히는 **판단·작문·해석 레이어**입니다:

| 구분 | 규칙 자동화 (Make/시나리오) | AI 직원 |
|---|---|---|
| 잘하는 일 | 정해진 시점에 정해진 문안 발송 | 상황 종합, 개인화 문안 작문, 우선순위 판단, 검수 |
| 예 | "D+5가 되면 경과체크 발송" | "오늘 뭐부터 해야 해?", "이 환자한테 뭐라고 보내지?" |
| 비용 | 건당 ~0원 | 업무 1건당 Claude 호출 2~8회 |

**핵심 안전 원칙 — AI는 발송 버튼을 누르지 않는다.**
모든 문안은 초안이며, 발송은 사람이 [개별 발송] 화면에서 확인 후 진행합니다.
(기획서 10.4 개인화 안전 규칙 — 오발송 1건이 '진심 케어'를 '소름 돋는 감시'로 뒤집는다)

## 2. 팀 구성 (오케스트레이터-워커 패턴)

```
        업무 지시 (자연어)
              │
              ▼
   ┌─────────────────────┐   데이터 조회 도구 (clinic_id 스코프 고정)
   │  🧑‍💼 AI 실장          │──▶ get_today_snapshot  오늘 내원·위기·체크포인트
   │  (오케스트레이터)      │──▶ search_patients     이름·차트번호 검색
   │                      │──▶ get_patient_360     환자 마스터+내원+프로그램+발송
   └──────────┬──────────┘
              │ ask_specialist(employee, task, context)
   ┌──────────┼──────────┬──────────────┐
   ▼          ▼          ▼              ▼
 ✍ 메시지   🛡 컴플라이언스  📈 데이터      📋 브리핑
   작가       검수관        분석가         비서
 (문안 초안)  (11항목 검수)  (숫자 해석)    (10초 브리핑)
```

| 직원 | 역할 | 산출물 |
|---|---|---|
| **AI 실장** | 업무 접수 → 데이터 조회 → 직원 배정 → 결과 종합. 데이터 조회 권한은 실장에게만 있음 | 최종 보고 |
| **메시지 작가** | 원장 명의 1인칭·담백한 문체로 개인화 케어 문안 초안. (광고) 표기·완곡 표현 기본기 내장 | 문안 + 채널 제안 |
| **컴플라이언스 검수관** | 의료법 27조 유인 / 의료광고 과장 / (광고) 표기 / 야간 / 완곡 표현 / 대상 적정성 / 내부 용어 노출 / 개인정보 과다 / **치료경험담·전후 비교 금지(56조)** / **의료광고 사전심의(57조) 대상 판정** / **부작용·개인차 고지** — 11항목 | ✅통과 / ⚠수정안 / ⛔불가 |
| **데이터 분석가** | 전달받은 숫자만 사용(지어내기 금지, n= 명시), 재내원·이탈 해석 | 핵심 발견 + 오늘의 실행 제안 |
| **브리핑 비서** | 진료 사이 2~3분용 — 임상 먼저, CRM 지표는 뒤 | 10초 환자/오늘 브리핑 |

**동작 흐름 예시** — "김민정님 재안내 문안 작성하고 검수까지":
실장이 `search_patients("김민정")` → `get_patient_360` → 작가에게 문안 위임 →
검수관에게 검수 위임 → 수정 반영한 최종본 + "발송은 사람이" 안내로 보고.

## 3. 배포 (4단계, ~10분)

전제: Supabase 연동 완료 상태 (`docs/SUPABASE-연동-가이드.md`).

1. **Anthropic API 키 발급** — [console.anthropic.com](https://console.anthropic.com) → API Keys → Create Key.
   결제 수단 등록 후 사용량 한도(Spend limit)를 걸어두는 것을 권장합니다.
2. **Edge Function 배포** — Supabase Dashboard → Edge Functions → Deploy a new function →
   이름 **`ai-employee`** → `db/edge-ai-employee.ts` 내용 붙여넣기 → Deploy.
3. **시크릿 추가** — Edge Functions → Secrets → **`ANTHROPIC_API_KEY`** = 발급받은 키.
4. **연동 테스트** — 대시보드 → AI 직원 → 연동 카드 [연동 테스트].
   (테스트는 모델 조회만 하므로 과금이 없습니다)

연동 전(또는 목데이터 모드)에는 화면이 **데모 시뮬레이션**으로 동작합니다 — 실제 AI 호출 없이
오케스트레이션 과정을 미리 볼 수 있습니다.

## 4. 기술 구성

- **모델**: `claude-opus-4-8` (파일 상단 `MODEL` 상수로 교체 가능) · 적응형 사고(adaptive thinking)
- **에이전트 루프**: 오케스트레이터가 도구 호출(`stop_reason === "tool_use"`)을 반복 처리, 왕복 상한
  `MAX_ROUNDS = 6` (엣지 함수 시간 보호). 병렬 도구 호출은 동시에 실행 후 한 메시지로 회신.
- **위임**: `ask_specialist` 도구 실행 = 전문 직원 시스템 프롬프트로 중첩 Claude 호출.
- **인증**: `send-sms`와 동일 — 로그인 사용자 + `clinic_members` 소속 확인. 모든 조회는 해당
  `clinic_id`로 고정.
- **응답**: `{ ok, employee, result, trace[], usage{input_tokens,output_tokens,calls} }` —
  trace(누가 무엇을 했는지)와 토큰 사용량이 화면에 그대로 표시됩니다.

### 개인정보 최소화 (기획서 8.2 준수)

- 환자 조회 컬럼에서 **전화번호(`phone`, `phone_norm`)와 발송토큰(`send_token`)을 제외** —
  AI에게 아예 전달되지 않습니다.
- 관여도 등급(상/중/하)은 내부 용어 — 환자 노출 문안에 쓰지 않도록 프롬프트에 명시.
- 발송중단·발송부적합·마케팅 미동의·위기 환자에 대한 광고성 제안 금지가 실장·검수관 양쪽에 내장.

## 5. 비용 가늠

업무 1건 = Claude 호출 2~8회 (직접 지명 1~2회, 오케스트레이션 3~8회).
결과 하단에 실제 호출 수·토큰이 표시되므로 며칠 써보고 실측하는 것이 가장 정확합니다.
대략: 브리핑·문안 1건당 수백 원 수준, 하루 10건 사용 시 월 수만 원 규모.
월 고정비 상한(기획서 9.3: 15만원) 관리를 위해 Anthropic 콘솔에서 Spend limit 설정을 권장합니다.

## 6. 한계와 다음 단계

**지금 안 되는 것(의도적)**
- AI가 직접 발송·DB 수정 — 읽기 전용 + 초안 생성만. 쓰기 권한은 사람의 버튼에만.
- 장시간 배치(전 환자 일괄 분석) — 엣지 함수 시간 제한. 필요 시 업무를 나눠 지시.

**로드맵 후보**
1. 아침 자동 브리핑 — Supabase Cron(pg_cron)으로 매일 08:30 오늘 브리핑을 생성해 큐 상단에 게시
2. 발송 검수 큐 연결 — 개별 발송 화면에서 [🛡 검수받기] 버튼으로 검수관 직접 호출
3. 문안 초안 → 메시지 템플릿 저장 원클릭
4. 관여도 재계산 검증 — 분석가가 점수 이력(DB5) 이상치 감지

## 7. 문제 해결

| 증상 | 원인·해결 |
|---|---|
| "ANTHROPIC_API_KEY 시크릿이 설정되지 않았습니다" | Edge Functions → Secrets에 키 추가 후 함수 재배포 |
| "ANTHROPIC_API_KEY가 올바르지 않습니다" | 키 오타/폐기 — 콘솔에서 재발급 |
| "사용량 한도에 걸렸습니다" | Anthropic 콘솔 사용량/한도 확인, 잠시 후 재시도 |
| "작업이 너무 복잡해 …" | 업무를 나눠 지시 ("오늘 브리핑" + "문안 작성"을 따로) |
| 버튼이 데모 시뮬레이션으로만 동작 | 환경설정에서 Supabase 연동(LIVE) 후 사용 가능 |
