# 튜브크루 — 유튜브 대행 AI 직원 가이드 (v0.1)

> 유튜브 대행사의 **기획 → 대본 → 편집 → 패키징** 제작 과정을 AI 직원 팀에게 맡기는 스튜디오입니다.
> 구현: `youtube-ai-studio.html` — 파일 하나로 동작하는 단일 HTML 앱 (설치 불필요, 브라우저에서 열면 끝).
> ※ 케어루프 CRM의 AI 직원(`docs/AI-직원-오케스트레이션-가이드.md`)과는 별개의 독립 도구입니다.

---

## 1. 팀 구성과 오케스트레이션 구조

```
채널 브리프 (채널명·타깃·톤·금지사항·주제·길이)
   │            ← 모든 직원이 이 브리프를 공통 컨텍스트로 받는다
   ▼
📌 기획 PD ──▶ ✍ 방송 작가 ──▶ 🎞 편집 디렉터 ──▶ 📦 패키징 매니저 ──▶ 📤 검수 실장
  기획안        풀 대본          편집 지시서         제목·썸네일·설명문      최종 교차 검수
   │             │                │                  │                 + 업로드 세팅 시트
   └───────── 🔍 검수 실장 (어느 단계든 [검수] 버튼으로 스팟 검수) ─────────────┘
   └───────── ✏ 수정 지시 (사람 피드백 → 해당 직원이 개정본 작성) ─────────────┘
```

| 직원 | 입력 | 산출물 |
|---|---|---|
| **📌 기획 PD** | 브리프 + 주제 | 한 줄 컨셉 · 훅 3안 · 타임라인 구성안 · 차별점 · 촬영 준비물 · 리스크 체크 |
| **✍ 방송 작가** | 기획안 | 타임코드 풀 대본 (3초/30초 훅, 오픈루프, 화면 지시, CTA, 예상 러닝타임) |
| **🎞 편집 디렉터** | 대본 | 컷 호흡 · 구간별 자막/B-roll/사운드 지시 · 챕터 · 저작권 체크리스트 · 컷 우선순위 |
| **📦 패키징 매니저** | 기획안+대본 | 제목 5안(패턴별) · 썸네일 문구 3안 · 설명문 · 태그 · 고정댓글 · 업로드 체크리스트 |
| **🔍 검수 실장 (스팟)** | 아무 산출물 | 8항목 검수(정책·노란딱지·저작권·협찬 고지·사실확인·낚시성·톤·초상권) → ✅/⚠/⛔ + 수정안 |
| **📤 검수 실장 (최종)** | 전 단계 산출물 | ①최종 교차 검수(제목-썸네일-본문 일치, 고지, 미해소 항목 → 게시 가능/조건부/보류) ②**업로드 세팅 시트** — 유튜브 스튜디오 순서 그대로: 확정 제목·설명문 전문·썸네일·재생목록·아동용(COPPA) 판단·유료 프로모션 체크·태그·카테고리·예약 공개 일시·종료화면/카드·고정댓글·게시 직후 30분 체크리스트 |

### 왜 "자유 오케스트레이터"가 아니라 "파이프라인 + 검수 게이트"인가

대행 제작은 단계와 승인 게이트가 정해진 워크플로입니다. 이런 일은 AI가 매번 순서를 판단하게
하는 것보다 **코드가 순서를 소유하고(파이프라인), AI는 각 단계의 전문 노동을 맡는** 편이
비용·품질·예측 가능성 모두에서 낫습니다. 대신 사람의 역할은 두 지점에 남습니다:

1. **단계 사이 승인** — 산출물을 직접 고치거나, '수정 지시'로 해당 직원에게 개정시킨 뒤 다음 단계로.
2. **검수 게이트** — 단계 중간엔 [🔍 검수]로 스팟 검수, 마지막엔 **📤 업로드 세팅 단계**에서
   검수 실장이 전 단계를 교차 검수하고 업로드 세팅 시트를 완성합니다. 미해소 항목이 있으면
   "게시 전 필수 해소" 목록과 함께 ⚠조건부/⛔보류로 판정합니다. **업로드 버튼은 사람이 누릅니다.**

전 단계를 한 번에 돌리고 싶으면 [🚀 전체 실행] — 기획→대본→편집→패키징→업로드 세팅이 순차
실행되고, 작업 로그에 누가 무엇을 했는지 트레이스가 남습니다.

## 2. 시작하기 (2분)

1. `youtube-ai-studio.html` 파일을 브라우저로 엽니다. **처음엔 데모 모드** — 실제 AI 호출 없이
   예시 산출물로 전체 흐름을 체험할 수 있습니다.
2. 실제 제작으로 전환: [console.anthropic.com](https://console.anthropic.com) → API Keys에서
   키 발급 → 스튜디오 우상단 **⚙ 설정** → 키 입력 → [연결 테스트].
   (테스트는 모델 정보 조회만 하므로 과금이 없습니다)
3. 채널 브리프를 채우고 [🚀 전체 실행] 또는 단계별 [🤖 맡기기].

### 잘 쓰는 요령

- **브리프가 품질의 8할** — 타깃("25~34 사회초년생, 투자 경험 없음")과 톤("어려운 용어 즉시
  풀어주기"), 금지사항("수익 보장 표현 금지")이 구체적일수록 납품 품질이 나옵니다.
- **수정 지시는 채팅처럼** — "훅을 더 자극적으로", "3번 구간 절반으로", "존댓말을 반말로".
  현재 버전 기준 개정본이 나오고, [↩ 이전 버전]으로 한 단계 되돌릴 수 있습니다.
- **산출물은 직접 고쳐도 됩니다** — 편집 내용은 자동 저장되고, 다음 단계 직원은 고친 버전을 받습니다.
- 프로젝트는 클라이언트/편 단위로 분리 — 상단 [＋ 새 프로젝트]. 모든 데이터는 이 브라우저
  (localStorage)에만 저장됩니다.

## 3. 기술 구성

- **모델**: `claude-opus-4-8`, 적응형 사고(adaptive thinking), `max_tokens 16000`
- **호출 방식**: 브라우저에서 Anthropic Messages API 직접 호출
  (`anthropic-dangerous-direct-browser-access` 헤더 — CORS 사전요청 확인 완료)
- **오케스트레이션**: 코드 소유 파이프라인 — 각 단계 산출물이 다음 단계 프롬프트에 컨텍스트로
  주입되는 체인. 수정 지시는 "현재 산출물 + 피드백 → 전체 개정본" 루프
- **사용량**: 프로젝트별 호출 수·토큰·대략 비용($)이 상단에 누적 표시
- **저장**: 프로젝트·산출물·로그 전부 localStorage — 서버 없음

### 보안 주의 (중요)

- API 키는 **이 브라우저에만** 저장되고 Anthropic 외 어디로도 전송되지 않습니다.
- 단, 브라우저 직접 호출 방식은 **본인 PC 개인 사용** 전제입니다. 파일을 팀원에게 공유하거나
  웹에 올릴 때는 반드시 ⚙ 설정에서 **키를 삭제**하고, 서버 프록시(예: Supabase Edge Function —
  `db/edge-ai-employee.ts` 패턴 재사용 가능) 방식으로 전환하세요.
- Anthropic 콘솔에서 **월 사용 한도(Spend limit)** 설정을 권장합니다.

## 4. 비용 가늠

- 업무 1건 = Claude 호출 1회. [🚀 전체 실행] = 5회, 검수·수정 지시는 각 1회.
- 영상 1편 풀 패키지(전체 실행 + 스팟 검수 1회 + 수정 3회 ≈ 9회) 기준 대략 $1~3 수준
  (분량·수정 횟수에 따라 변동). 정확한 값은 상단 사용량 표시로 실측하세요.

## 5. 한계와 다음 단계

**지금 안 되는 것(의도적)**
- 영상 파일 편집·업로드 — AI는 지시서와 텍스트 산출물까지. 컷 편집과 업로드 버튼은 사람.
- 트렌드 실시간 리서치 — 현재 브리프 기반 기획. (다음 단계 후보: 웹 검색 도구 연결)

**로드맵 후보**
1. 기획 PD에 웹 검색 연결 — 경쟁 영상·트렌드 리서치 기반 기획안
2. 대본 → 자막 파일(SRT) 자동 변환
3. 원본 영상 타임라인(촬영본 로그) 입력 → 실제 타임코드 기반 컷 지시서
4. 클라이언트 공유용 읽기 전용 납품 페이지 내보내기
5. 팀 공유가 필요해지면: Supabase Edge Function 프록시 + 계정별 권한

## 6. 문제 해결

| 증상 | 원인·해결 |
|---|---|
| "API 키가 올바르지 않습니다" | 키 오타/폐기 — 콘솔에서 재발급 후 ⚙ 설정에서 교체 |
| "사용량 한도 초과" | Anthropic 콘솔에서 한도·크레딧 확인 |
| 산출물이 중간에 잘림 | 목표 길이가 길 때 발생 가능 — "이어서 계속"으로 수정 지시하거나 단계를 나눠 지시 |
| 편집 지시서 버튼이 안 눌림 | 대본이 먼저 있어야 합니다 (파이프라인 순서 보호) |
| 업로드 세팅 버튼이 안 눌림 | 패키징이 먼저 있어야 합니다 — 세팅 시트는 패키징 산출물을 확정하는 단계 |
| 데모 모드에서 안 벗어남 | ⚙ 설정에서 [연결 테스트]까지 눌러야 키가 저장됩니다 |
