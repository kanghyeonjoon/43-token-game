# 튜브크루 — 유튜브 대행 AI 직원 가이드 (v0.1)

> 유튜브 대행사의 **기획 → 대본 → 편집 → 패키징** 제작 과정을 AI 직원 팀에게 맡기는 스튜디오입니다.
> 구현: `youtube-ai-studio.html` — 파일 하나로 동작하는 단일 HTML 앱 (설치 불필요, 브라우저에서 열면 끝).
> ※ 케어루프 CRM의 AI 직원(`docs/AI-직원-오케스트레이션-가이드.md`)과는 별개의 독립 도구입니다.

---

## 1. 팀 구성과 오케스트레이션 구조

```
채널 브리프 = 대표 가이드라인 (채널·타깃·톤·금지사항·주제·길이)
   │
   ├─ 💡 주제 발굴 (선택) — 주제가 없거나 애매하면 기획 PD × 패키징 매니저 아이템 미팅
   │     후보 6개(유형이 전부 다름) 제출 → 대표가 고르면 주제 확정 · 피드백으로 재발굴
   ▼
🗣 제작 킥오프 회의 — AI 직원 5명이 서로의 발언을 읽고 반응하며 방향 합의
   기획 PD(방향 제안) → 작가(대본 관점 반박·요구) → 편집(실현 가능성) → 패키징(CTR 앵글) → 검수(가드레일)
   │
   ▼
📝 제작 방향 합의서 — 기획 PD가 회의록 종합 (미결 사항은 대표가 결정 · 직접 수정 가능)
   │            ← 이후 전 단계가 이 합의서를 기준으로 일한다 (재작업 방지)
   ▼
📌 기획 PD ──▶ ✍ 방송 작가 ──▶ 🎞 편집 디렉터 ──▶ 📦 패키징 매니저 ──▶ 📤 검수 실장
  기획안        풀 대본          편집 지시서         제목·썸네일·설명문      최종 교차 검수
   │             │                │                  │                 + 업로드 세팅 시트
   └───────── 🔍 검수 실장 (어느 단계든 [검수] 버튼으로 스팟 검수) ─────────────┘
   └───────── ✏ 수정 지시 (사람 피드백 → 해당 직원이 개정본 작성) ─────────────┘
```

### 💡 주제 발굴 — 주제부터 팀에게 맡기기

"주제를 던져줬는데 주제부터 별로"인 경우를 위한 앞단입니다. 브리프의 [💡 주제 발굴] 버튼을
누르면 **차민준 PD(트렌드·시청자 가치 관점)와 한유진 매니저(화제성·클릭 각 관점)가 아이템
미팅**을 열어, 두 관점을 모두 통과한 후보 6개를 냅니다. 후보 6개는 유형이 전부 다릅니다 —
절차형 / 실수·함정형 / 비교·검증형 / 스토리형 / 시의성형 / 리스트형. 각 후보에는 📌 왜 지금
이 채널인가(PD) · 📦 클릭을 만드는 각(패키징) · 🔍 주의점(검수)이 붙습니다.
[이 주제로]를 누르면 영상 주제 칸에 바로 들어가고, 마음에 안 들면 피드백을 적어
[🔄 다시 발굴]하면 그 방향으로 전부 다시 냅니다. 주제 메모를 미리 적어뒀다면 그걸 발전시키거나
더 나은 각을 제안합니다. 호출 1회라 부담 없이 여러 번 돌려도 됩니다.

### ⏹ 진행 중지

회의·전체 실행·단계 작업·검수·주제 발굴이 도는 동안 [⏹ 중지] 버튼이 나타납니다. 누르면
진행 중인 API 호출을 즉시 끊고 멈춥니다 — 회의는 지금까지의 발언이 회의록에 남고, 전체 실행은
완료된 단계 산출물이 그대로 남습니다(중단된 단계의 기존 산출물도 유지). 중지 후 언제든
다시 시작할 수 있습니다.

### 🏢 오피스 뷰 — AI 직원들이 일하는 가상 사무실

화면 상단에 픽셀 캐릭터 5명이 있는 **가상 사무실**이 표시됩니다 (탑바 [🏢 오피스]로 켜고 끔).
각자 자기 방(기획실·대본실·편집실·패키징룸·검수실) 책상에 있다가:

- **회의가 시작되면 전원이 중앙 회의 테이블로 걸어와** 앉고, 발언이 캐릭터 머리 위
  말풍선으로 실시간 스트리밍됩니다. 회의가 끝나면 각자 자리로 돌아갑니다.
- **작업 중**엔 담당 캐릭터가 책상에서 들썩이며(상태점 주황) 쓰고 있는 산출물이 말풍선으로
  흐르고, 끝나면 **완료 ✓**(초록), 실패하면 **막혔어요!**(빨강)가 뜹니다.
- 주제 발굴 때는 차민준 PD와 한유진 매니저 둘만 테이블로 나와 아이템 미팅을 합니다.
- 상단 칩에 작업중 인원·산출물 진행(n/6)·호출·토큰·비용이 실시간 집계되고,
  **캐릭터를 클릭하면 그 직원의 담당 단계 탭으로 이동**합니다.

캐릭터는 외부 이미지 없이 인라인 SVG로 그려서 단일 HTML 파일 그대로입니다.

### 🔎 웹 리서치 — 감이 아니라 검색으로 기획

Anthropic **서버 사이드 웹 검색 도구**(`web_search_20260209`)가 연결돼 있습니다 (라이브 전용):

- **기획안 탭의 [🔎 웹 리서치]** 를 켜고 맡기면, 차민준 PD가 같은 주제 경쟁 영상·최신 트렌드·
  인용 수치를 웹에서 3~5회 검색한 뒤 기획안을 씁니다. '차별점'이 실제 경쟁 콘텐츠 대비로 작성되고,
  기획안 끝에 [리서치 출처] 목록이 붙습니다.
- **주제 발굴의 [🔎 트렌드 검색]** 을 켜면 후보 6개가 실제 최신 이슈·화제 소재를 반영해 나옵니다.
- 검색이 길어지면(`pause_turn`) 자동으로 이어서 진행합니다. 비용: 검색 $10/1,000회 + 늘어난 토큰 —
  매번 켜기보다 새 주제·시의성 주제에서 켜는 걸 권합니다. 데모 모드에서는 일반 모드로 동작해요.

### 🎓 직원 증원 — 채용 가능한 직원

팀 카드 아래 **채용 버튼**으로 직원을 늘릴 수 있습니다 (언제든 퇴사 가능, 기록 유지):

| 직원 | 담당 | 🧠 빌려온 뇌 |
|---|---|---|
| **🎨 오하늘 · 썸네일 디자이너** | 썸네일 시안·문구, 촬영 스틸 지시 | 미스터비스트 썸네일 원칙(상황 1개+감정) · 모바일 퍼스트(작게 봐도 읽히는가) · 오길비(이미지도 약속) |
| **💬 강태오 · 커뮤니티 매니저** | 고정댓글·커뮤니티 탭·악플/스팸 대응 | 세스 고딘 부족(Tribe) · 커뮤니티 운영 실무(초기 1시간) · 브랜드 세이프티 |

채용하면 **회의에 참석**해 발언하고(호출 +1회/명), 오피스에 방(디자인실·커뮤니티실)이 생기며,
업무 보드 자동 배정이 우선 매칭됩니다(썸네일·시안→오하늘, 댓글·공지·팬→강태오).
캐릭터를 클릭하면 업무 보드로 연결돼요.

### 📥 업무 보드 — 회의·파이프라인과 별개로 일 던지기

급한 업무("썸네일 문구 5개만 더", "이 대본 심의 리스크만 봐줘")를 입력하고 [던지기]를 누르면
큐에 쌓이고, **직원이 한가해질 때 순서대로 집어갑니다** — 오피스에서 담당 캐릭터가 일하는 게
보이고, 항목을 누르면 결과를 확인·복사할 수 있어요.

- **자동 배정**: 썸네일·제목·설명문·태그 → 한유진 / 대본·멘트·훅 → 윤세아 / 편집·자막·컷·BGM → 박도현 /
  검수·심의·리스크·저작권 → 서정우 / 그 외 → 차민준. 셀렉트로 직접 지정도 가능.
- 처리 기준: 채널 브리프 + (있다면) 제작 방향 합의서. 자유 업무 1건 = 호출 1회.
- 회의·파이프라인이 도는 동안엔 대기하고, 끝나면 이어서 처리합니다. 중단된 업무는
  결과 창에서 [🔁 다시 대기로]로 재실행.

### ⏸ 배속 · 일시정지

오피스 헤더에 **배속 칩(0.5× / 1× / 2× / 4×)** 과 **[⏸ 일시정지]** 가 있습니다.
데모 모드는 즉시(타자 효과 중간에도) 멈추고 배속되며, 라이브 모드는 진행 중인 API 호출을
마친 뒤 다음 스텝(다음 발언·다음 단계·다음 업무)부터 멈춥니다. 배속은 저장되어 유지됩니다.

### 📋 기획안은 전용 창에서 · 💾 채널은 저장해서 재사용

- **[📋 기획안·자료 넣기]** — 브리프의 기획안 버튼을 누르면 큰 편집 창이 열립니다.
  붙여넣기 또는 **.txt/.md 파일 열기**로 통째로 첨부하면, 팀이 그 자료를 '기준'으로 회의하고
  기획 PD는 뒤집는 게 아니라 검토·보강합니다 (기존 대표 자료 로직 그대로).
- **[💾 채널 저장]** — 채널명·카테고리·타깃·톤·금지사항·협찬을 저장해두면, 새 프로젝트에서
  '저장 채널' 선택만으로 6개 필드가 자동으로 채워집니다. 매번 다시 적을 필요 없이
  **주제와 기획안만** 새로 넣으면 돼요. (이 브라우저 localStorage에 저장)

### 🗣 회의는 실시간으로 보입니다 (스트리밍)

발언이 끝난 뒤 한 번에 뜨는 게 아니라, **말하는 대로 타이핑되듯 흘러나옵니다** — 라이브 모드는
Anthropic API의 SSE 스트리밍으로 실제 생성 중인 토큰을 그대로 표시하고, 데모 모드도 같은
타자 효과로 동작합니다. 합의서·기획안·대본 등 산출물 작성과 검수, 주제 발굴(후보가 하나씩 등장)도
전부 실시간입니다. 진행 중 언제든 [⏹ 중지]로 끊을 수 있습니다.

- **🔁 반박 라운드 (선택)** — 회의 탭의 체크박스를 켜면, 전원 발언 후 한 바퀴 더 돌며 서로의
  발언을 반박하거나 자기 안을 수정합니다 (호출 +5회). 쟁점이 갈리는 주제에 쓰세요.
- **🗂 회의록 히스토리** — 재회의하면 이전 회의록이 보존되고, [🗂 이전 회의록] 버튼으로
  비교해볼 수 있습니다.
- **📊 스코어 추이** — 재회의 시 스코어보드에 직전 회의 대비 ▲▼ 변화가 표시됩니다.
- **판정 배지** — 업로드 세팅 탭 상단에 검수 실장의 최종 판정(✅ 게시 가능 / ⚠ 조건부 / ⛔ 보류)이
  배지로 승격되어 한눈에 보입니다.

### 제작 회의가 왜 필요한가

단계를 따로따로 진행하면 뒤 단계에서 문제가 발견됐을 때 앞 단계부터 다시 해야 합니다
(예: 패키징에서 "이 제목 각도면 훅을 바꿔야 한다" → 대본·편집 재작업). 킥오프 회의에서는
각 직원이 **자기 단계에서 재작업을 만들 위험을 미리 발언**합니다 — 작가는 "기획안에 구간별
목적이 없으면 내가 다시 정하게 된다", 편집은 "훅에 체결 화면을 쓰려면 촬영 순서를 바꿔야
한다", 검수는 "제목의 '12분'은 실측 못 하면 약속 위반"처럼. 합의된 방향은 **제작 방향
합의서**로 문서화되고, 이후 모든 단계 프롬프트에 "이 방향을 벗어나지 말 것"으로 주입됩니다.
회의에서 결론이 안 난 쟁점은 **미결 사항**으로 올라와 대표(사람)가 결정합니다.

| 직원 | 🧠 빌려온 뇌 (판단 기준) | 산출물 |
|---|---|---|
| **📌 차민준 · 기획 PD** | 미스터비스트(클릭 못 참는 컨셉 먼저·첫 1분 증명) · 패디 갤러웨이(아이디어=패키징 동시 설계, "지금 왜") · 나영석(착한 재미·시리즈 확장성) | 한 줄 컨셉 · 훅 3안 · 타임라인 구성안 · 차별점 · 리스크 체크 |
| **✍ 윤세아 · 방송 작가** | 케이시 나이스탯(정보도 스토리로·페이싱) · 로버트 맥키(구간마다 시청자 상태 변화) · 이우정(말맛·캐릭터 일관성) | 타임코드 풀 대본 (훅, 오픈루프, 화면 지시, CTA, 러닝타임) |
| **🎞 박도현 · 편집 디렉터** | 월터 머치(감정>스토리>리듬 컷) · 미스터비스트 편집실(리텐션 그래프 사고) · 예능 자막 문법(자막=제2의 화자) | 컷 호흡 · 자막/B-roll/사운드 지시 · 챕터 · 저작권 체크리스트 |

### 🏥 의료 콘텐츠(병원·의원 채널) 특별 규정 — 의료광고법 준수

**의료법 56·57조(의료광고 규제)** 를 최우선으로 적용합니다:

- 치료경험담(환자 후기·인터뷰) 형식 광고 금지 · 치료 전후 사진/비교 금지
- 치료효과 보장·오인 표현 금지 ("완치", "부작용 없음", "최초·최고")
- 다른 의료기관·의료인 비교·비방 금지 · 환자 유인(할인·이벤트, 의료법 27조) 금지
- 부작용·개인차 고지 누락 확인
- **사전심의**: 의료기관이 유튜브 등 온라인 매체에 올리는 광고성 콘텐츠는 자율심의기구
  (의사회·치과의사회·한의사회 등) 사전심의 대상일 수 있음 — 검수 판정에 심의 필요 여부가
  별도 줄로 표시되고, 의료 콘텐츠로 판단되면 판정 첫 줄에 "🏥 의료광고 규정 적용"이 붙습니다.

같은 제약이 전 직원 공통 원칙에도 들어가 있어(작가는 후기 형식·전후 비교를 초안부터 쓰지 않음),
검수에서 뒤집히기 전에 예방됩니다. ※ 케어루프 CRM의 컴플라이언스 검수관에도 동일 조항이 반영되어 있습니다.

### 논의 3축과 팀 평가 스코어보드 (시각화)

회의에서는 모든 참석자가 **①트렌드(지금 왜 이 주제인가) ②콘텐츠 가치(시청자가 얻는 것)
③화제성(공유·댓글을 부르는 힘)** 중 자기 담당과 맞닿은 축을 의무적으로 짚습니다. 회의가 끝나면
기획 PD가 합의서와 함께 **팀 자평 점수**(트렌드 적합성/콘텐츠 가치/화제성/실행 가능성/안전성,
각 0~10)를 내고, 화면에 **스코어보드(총점 + 축별 바 차트)** 로 표시됩니다. 4점 이하 축이 있으면
보완(재논의·합의서 수정) 후 제작을 권장하는 경고가 붙습니다.

또한 팀이 **실제로 일하는 게 보이게** 되어 있습니다: 회의 중에는 발언자의 "생각을 정리해 발언하는
중…" 입력 표시가 뜨고, 좌측 팀 카드에는 각 직원의 실시간 상태(발언 중/기획안 작성 중/검수 중/대기)가
표시됩니다. 회의록은 아바타·이름이 있는 채팅으로 남습니다.

### 왜 "자유 오케스트레이터"가 아니라 "파이프라인 + 검수 게이트"인가

대행 제작은 단계와 승인 게이트가 정해진 워크플로입니다. 이런 일은 AI가 매번 순서를 판단하게
하는 것보다 **코드가 순서를 소유하고(파이프라인), AI는 각 단계의 전문 노동을 맡는** 편이
비용·품질·예측 가능성 모두에서 낫습니다. 대신 사람의 역할은 두 지점에 남습니다:

1. **단계 사이 승인** — 산출물을 직접 고치거나, '수정 지시'로 해당 직원에게 개정시킨 뒤 다음 단계로.
2. **검수 게이트** — 단계 중간엔 [🔍 검수]로 스팟 검수, 마지막엔 **📤 업로드 세팅 단계**에서
   검수 실장이 전 단계를 교차 검수하고 업로드 세팅 시트를 완성합니다. 미해소 항목이 있으면
   "게시 전 필수 해소" 목록과 함께 ⚠조건부/⛔보류로 판정합니다. **업로드 버튼은 사람이 누릅니다.**

전 단계를 한 번에 돌리고 싶으면 [🚀 전체 실행] — 기획→대본→편집→패키징→업로드 세팅이 순차
실행되고, 작업 로그에 누가 무엇을 했는지 트레이스가 남습니다.

### 🧭 페이블 모드 — fablize 규율 이식

[fablize](https://github.com/fivetaku/fablize)의 요지는 "하네스는 모델의 천장을 올릴 수 없지만,
**절차적 규율로 모델이 자기 천장까지 가게 만들 수는 있다**"입니다. 그중 이전 가능한 규율을
전 직원 프롬프트에 이식했습니다:

- **증거 없는 완료 금지 + [자가 검증] 게이트** — 모든 산출물 끝에 완료 조건을 목표로 분해하고
  항목별 ✅(증거: 어느 부분) / ❌(미충족 사유)를 표기. ❌가 하나라도 있으면 '완료'라 말하지 못합니다
- **조기 중단 방지** — "~하면 됩니다 / ~할 예정"으로 끝내기 금지. 지금 이 산출물 안에서 실제로 작성
- **수치 검증** — 글자수·러닝타임·제목 자수처럼 셀 수 있는 조건은 실제로 세어 수치를 증거로 표기
- **조사 프로토콜(검수)** — 지적마다 '위반이 아닐 수도 있는' 경쟁 해석을 검토한 뒤 판정하고
  기각 근거를 남깁니다 (과잉 검열 방지 + 원인을 고치는 수정안)
- **합의서 근거화** — 합의서 각 항목 끝에 (근거: 발언자) 표기, 회의록에 근거 없는 항목 금지
- **능력 한계 보고** — 같은 지적이 수정 2회 이상 반복되면, 막힌 지점과 사람이 결정할 대안 2개를 보고

모델은 그대로 `claude-opus-4-8` — **비용 증가 없이 규율만 이식**됩니다. 모델 자체를 올리려면
`MODEL` 상수 한 줄을 `claude-fable-5`로 교체하세요 (비용 약 2배: 입력 $10 / 출력 $50 per 1M 토큰,
30일 데이터 보존 설정 필요).

### 잘 쓰는 요령

- **브리프가 품질의 8할** — 타깃("25~34 사회초년생, 투자 경험 없음")과 톤("어려운 용어 즉시

- **이미 기획안이 있다면 '대표 자료' 칸에 통째로 붙여넣기** — 회의가 그 자료의 검토·보강으로
  진행되고(대표 안 갈아엎기 금지 규칙), 합의서에 "7-1) 대표 안에서 바꾸기로 한 것"이 근거와 함께
  표기되며, 기획 PD는 대표 안의 뼈대를 유지한 '실행 기획안'을 만듭니다 (다르게 보는 부분은
  [제안] 표시). 완성 기획안을 그대로 쓰고 싶으면 기획안 탭의 산출물 칸에 직접 붙여넣으세요 —
  대본부터의 모든 단계가 그 버전을 받습니다.
- **수정 지시는 채팅처럼** — "훅을 더 자극적으로", "3번 구간 절반으로", "존댓말을 반말로".
  현재 버전 기준 개정본이 나오고, [↩ 이전 버전]으로 한 단계 되돌릴 수 있습니다.
- **방향 자체가 바뀌면 회의를 다시** — 합의서의 잔손질은 직접 수정이나 '수정 지시'(기획 PD 단독,
  1회 호출)로, 큰 방향 전환은 [🗣 제작 회의] 재실행(기존 합의서를 재검토 대상으로 올려 전원 재논의).
- **산출물은 직접 고쳐도 됩니다** — 편집 내용은 자동 저장되고, 다음 단계 직원은 고친 버전을 받습니다.
- 프로젝트는 클라이언트/편 단위로 분리 — 상단 [＋ 새 프로젝트]. 모든 데이터는 이 브라우저
  (localStorage)에만 저장됩니다.

## 3. 기술 구성

- **모델**: `claude-opus-4-8`, 적응형 사고(adaptive thinking), `max_tokens 16000`
- **호출 방식**: 브라우저에서 Anthropic Messages API 직접 호출
  (`anthropic-dangerous-direct-browser-access` 헤더 — CORS 사전요청 확인 완료)
- **스트리밍**: `stream: true` SSE — `content_block_delta`(text_delta)로 발언·산출물을 실시간 렌더,
  `message_start`/`message_delta`에서 토큰 사용량·stop_reason 수집. 중지는 `AbortController`로 스트림 즉시 차단
- **오케스트레이션**: 코드 소유 파이프라인 — 각 단계 산출물이 다음 단계 프롬프트에 컨텍스트로
  주입되는 체인. 수정 지시는 "현재 산출물 + 피드백 → 전체 개정본" 루프
- **사용량**: 프로젝트별 호출 수·토큰·대략 비용($)이 상단에 누적 표시
- **저장**: 프로젝트·산출물·로그 전부 localStorage — 서버 없음

### 보안 주의 (중요)

- API 키는 **이 브라우저에만** 저장되고 Anthropic 외 어디로도 전송되지 않습니다.
- 단, 브라우저 직접 호출 방식은 **본인 PC 개인 사용** 전제입니다. 파일을 팀원에게 공유하거나
  웹에 올릴 때는 반드시 ⚙ 설정에서 **키를 삭제**하고, 서버 프록시(예: Supabase Edge Function —
  `db/edge-ai-employee.ts` 패턴 재사용 가능) 방식으로 전환하세요.
- Anthropic 콘솔에서 **월 사용 한도(Spend limit)** 설정을 권장합니다.

## 4. 비용 가늠

- 업무 1건 = Claude 호출 1회. [🗣 제작 회의] = 발언 인원+1회(기본 6회 · 채용 시 인원만큼 증가 ·
  🔁 반박 라운드 켜면 발언 ×2), [🚀 전체 실행] = 5회, [💡 주제 발굴]·검수·수정 지시·📥 자유 업무는 각 1회.
  🔎 웹 리서치는 검색 비용($10/1,000회)과 조사 토큰이 추가됩니다.
  [⏹ 중지]하면 그 시점까지 완료된 호출만 과금됩니다 (스트리밍 중 중단 시 이미 생성된 토큰은 과금).
- 영상 1편 풀 패키지(회의 + 전체 실행 + 스팟 검수 1회 + 수정 3회 ≈ 15회) 기준 대략 $2~5 수준
  (분량·수정 횟수에 따라 변동). 정확한 값은 상단 사용량 표시로 실측하세요.
