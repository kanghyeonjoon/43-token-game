# Vrew 클론 검증 리포트

> 질문: "유튜브 영상(소프트웨어의 종말…)에서 말한 방식으로 Vrew 같은 자막 기반 영상편집 툴을 만들 수 있는가?"
> 결론: **가능하다.** 핵심 기능(AI 자동 자막, 단어 컷 편집, 무음 자동 컷, 자막 수정, 내보내기)을 브라우저 단일 HTML 파일로 구현하고 E2E 테스트 51개로 실동작을 검증했다.

## 결과물

- **`vrew-editor.html`** — 서버 없이 브라우저에서 바로 열어 쓰는 단일 파일 앱. 영상 파일은 업로드되지 않고 전부 로컬(브라우저 메모리)에서 처리된다.
- 열자마자 데모 프로젝트(그릭 요거트 브이로그 대본 7클립)가 로드되어 바로 체험 가능.

## 구현된 기능 (전부 실동작, E2E 검증됨)

| 기능 | 방식 |
|---|---|
| **AI 자동 자막 (음성 인식)** | 자막 탭. ① 브라우저 로컬: transformers.js + Whisper(tiny/base/small) — 키 불필요, 음성이 밖으로 안 나감, 최초 1회 모델 다운로드 후 캐시 ② Whisper API: OpenAI 호환 서버(Groq 등 포함), 16kHz WAV로 변환해 전송, 단어 타임스탬프(verbose_json) 파싱. 인식된 단어는 휴지(0.6초)·문장부호·12단어 상한 기준으로 클립 자동 분할 |
| 단어 단위 컷 편집 | 단어 칩 선택 → Delete. 컷된 단어의 영상 구간을 재생·내보내기에서 건너뜀 |
| 클립 분할 / 병합 / 전체 컷 | 편집 탭. 실행취소/다시실행(Ctrl+Z/Y) 지원 |
| 자막 오타 수정 | 💬 줄 클릭 편집 → 글자 수 비례로 단어 타이밍 재배분 |
| 무음 구간 자동 컷 | Web Audio API로 오디오 디코딩 → 50ms RMS 분석 → 0.4초 이상 무음 자동 컷 |
| 영상/자막 불러오기 | mp4·webm + SRT/VTT. SRT 없으면 대본 붙여넣기로 클립 생성 |
| 자막 서식 | 크기·색·외곽선·배경상자·위치 (서식 탭) |
| 무료 애셋 패널 | 검색·태그·카테고리 필터, 클릭으로 스티커 삽입, 드래그 이동 |
| 내보내기 | ① SRT(컷 반영 타임라인 재계산) ② 편집본 WebM(캔버스+MediaRecorder 실시간 렌더, 자막 번인) ③ 편집툴 XML(xmeml v4 — Premiere Pro·DaVinci Resolve 가져오기, 컷=시퀀스 클립·자막=마커) ④ 컷 데이터 JSON(keep_segments — 타 툴 이식용) ⑤ 프로젝트 저장/열기 |

## 검증 방법 (영상에서 말한 "테스트까지 AI에게" 그대로)

Playwright(Chromium 헤드리스) E2E 4개 스위트, 총 **65개 체크 전부 통과**:

1. **데모 경로 (23개)** — 클립 로드, 단어 클릭 시크, 컷 후 재생 스킵, SRT 타임라인 재계산, 분할/병합/실행취소, 자막 수정 재배분, 애셋 검색/삽입, 서식 반영, 컷 JSON 연속성, WebM 다운로드.
2. **실영상 경로 (8개)** — 브라우저에서 생성한 8초 테스트 영상(톤/무음 교차)으로: 영상 로드, SRT 로드, 무음 구간 2개 자동 감지(2.6~3.9s, 6.6~7.9s), 재생 중 무음 스킵, 영상+오디오 WebM 내보내기.
3. **AI 자막 경로 (20개)** — 단어→클립 그룹핑 규칙(휴지/문장부호/12단어/null 타임스탬프 보정), Whisper API 전체 플로우(모의 서버로 요청 형식·Bearer 키·multipart·응답 파싱·클립 생성·실행취소 검증), 401 오류 시 우아한 실패, 브라우저 로컬 경로의 transformers.js 실물(npm v3.3.3) 동적 로드.
   ※ 로컬 Whisper의 실제 모델 추론은 이 개발 환경의 네트워크 정책(HuggingFace 차단)으로 실행 불가 — 라이브러리 로드와 실패 처리까지만 검증했고, 실제 인식 품질은 사용자 브라우저(인터넷 연결)에서 확인 필요.
4. **XML 내보내기 (14개)** — DOMParser·Python 파서 이중 검증, 컷 1회 → V/A 클립 2개, 소스 in/out 프레임 정확도, 타겟 타임라인 연속성, 시퀀스 길이=편집본 프레임, 자막 마커 7개, 특수문자(`<>&"`) 이스케이프, UI 버튼 실다운로드.
   ※ 실제 Premiere/Resolve 임포트는 이 환경에 해당 프로그램이 없어 미검증 — xmeml v4 규격 준수까지 확인.

테스트 중 발견해 수정한 실버그 3건:
- 자막 줄 클릭 시 목록 재렌더링으로 편집 포커스 소실
- `display:none` 캔버스에서 프레임 캡처가 생략돼 WebM이 0바이트
- MediaRecorder 녹화 webm의 duration이 `Infinity`로 읽혀 내보내기가 끝나지 않음 (끝으로 시크해 길이 보정)

## 한계 (원조 Vrew 대비)

- **로컬 Whisper 정확도는 모델 크기에 비례** — tiny/base는 한국어 인식 품질이 낮을 수 있어 base 이상 권장. 인식은 CPU(WASM)라 영상 길이에 따라 수 분 소요. 정확도가 중요하면 API 모드 사용.
- **WebM 내보내기는 실시간 녹화** — 편집본 길이만큼 시간이 걸린다. ffmpeg.wasm을 쓰면 오프라인 렌더 가능.
- **캡컷 내보내기는 간이 포맷** — 실제 CapCut draft_content.json 규격이 아니라 keep_segments/자막 목록 JSON. 규격 리버스 엔지니어링하면 확장 가능.

## 사용법

`vrew-editor.html`을 브라우저(Chrome 권장)에서 열기 → 도움말 탭 참고.
